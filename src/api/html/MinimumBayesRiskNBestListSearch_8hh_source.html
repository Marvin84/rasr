77 /**
78  * Implementation of the naive approach for minimum Bayes risk using an nbestlist for both
79  * the evaluation space as well as for the summation space.
80  * The search can be performed on two different nbestlists depending on how much hypotheses
81  * shall be used for minimization or summation. In order to stay conform to the MinimumBayesRiskSearch
82  * interface, the search itself will be given a single nbestlist from which the two lists
83  * for evaluation space and summation space will be extracted up to a configurable number
84  * of string hypotheses.
85  * In addition there exist two criteria which can abbreviate the search process. Their application
86  * can be configured. If fulfilled, this information will be logged to the information channel.
87  * It should be noted that the computation of the two criteria do not afford additional computation
88  * time but come along with the computation process.
89  *
90  */
