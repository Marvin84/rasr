24 /**
25  * Normalize the state ids of an automaton. With the "destructive"
26  * algorithms, e.g. trimming, pruning, it happens that the sequence of
27  * state ids get holes. As some other algorithms allocate vectors based
28  * on state ids as index this may result in poor memory efficiency.
29  * This function removes all holes in state ids on-demand by renumbering
30  * them in the order they are requested. We found out empirically
31  * that mapping the indices on-demand is quicker and more memory
32  * efficient than using hash tables instead of vectors.
33  * Category: on-demand
34  * @param f the input automaton
35  * @return returns an automaton with all state ids being normalized
36  **/
37 template<class _Automaton>
38 typename _Automaton::ConstRef normalize(typename _Automaton::ConstRef f);
