33 /**
34  * @page Channels
35  *
36  * Instead of using cout and cerr directly you should use
37  * channels. Channels are named output streams that can be
38  * configured and redirected individually. Channels also take
39  * care of character set conversion and (optionally) XML
40  * formatting. (Using cerr is only allowed in those places of
41  * Core where the channel sub-system has not been set up yet.
42  *
43  * Channels cause NO (!) overhead compared to standard streams if
44  * you use only a single destination stream. Multiple destination
45  * streams through redirection have the overhead of making multiple
46  * copies of the data to be written (conversion to character strings
47  * occurs only once!)
48  *
49  * Upon creation a Channel determines its set of targets via the
50  * standard configuration process. E.g. if your component is
51  * called "recognizer" then Channel(config, "statistics"), will
52  * look for a resource matching "recognizer.statistics.channel".
53  * The resource's value is interpreted as a comma separated list
54  * of target names, specifying were the channel's data is sent.
55  *
56  * The following channel targets are predefined:
57  * -# "stdout" standard output of process
58  * -# "stderr" standard error output of process
59  * -# "nil" suppress output
60  *
61  * All other target names cause the channel manager to create
62  * additional targets as needed. Each target is a Configurable
63  * registered under "<application-title>.channels.<target-name>".
64  * If the target name contains a dot, only the part after the dot
65  * is used as parameter name. By default a channel target will
66  * open a file by its own name and write all output to this
67  * file. This can be overridden with the "file" parameter.
68  *
69  * Channel targets can be configured in several ways:
70  * -# The file name can be changed to something different from the
71  * target name using the "file" parameter.
72  * -# If the file already exists it is overwritten by default.
73  * By setting "append" to true, the channel manager will append
74  * to the file instead of overwriting.
75  * -# File output is buffered by default, which can cause long
76  * delay in the output. Set "unbuffered" to change to
77  * line-buffering mode.
78  * -# Internally channels expect to be provided with UTF-8 encoded
79  * character data. (Plain ASCII is a subset of UTF-8.) It is
80  * possible to specify a specific output encoding for each channel
81  * target, by setting the parameter "encoding" (ISO-8859-1 by
82  * default). The channel will convert the data into this
83  * character set encoding upon output.
84  * -# Channels feature automatic word-wrapping, which is disabled
85  * by default. To enable word-wrapping, set the parameter
86  * "margin" to the number of characters per line.
87  * -# XML (and possibly other) text is automatically indented.
88  * The parameter "indentation" controls the depth of indentation.
89  * Naturally, a value of zero, disable auto-indentation.
90  * -# zlib compression can be activated using the "compressed" parameter.
91  * The filename will be extended by the suffix ".gz" if not already
92  * present
93  *
94  * You can check whether a channel's output is actually used by calling
95  * isOpen(). Make use of this especially if your output needs additional
96  * calculations.
97  *
98  * For plain text output use Channel. If you want to produce XML
99  * output, use the derived class XmlChannel.
100  **/
