46 /**
47  * Add language model scores to a Fsa automaton.
48  *
49  * This is theoretically almost equivalent to using Fsa::compose()
50  * with the FSA representation of the language model obtained from
51  * getFsa(), but there are the following differences:
52  * - This function works for any LanguageModel, even if it does not
53  * implement getFsa().
54  * - Since the LanguageModel interface presents a deterministic
55  * epsilon-free automaton, this alogrithm is simpler.
56  * - No back-off paths are included. So the result is also correct
57  * if getFsa() suffers from the back-off arc shortcut phenomenon.
58  * - As a corrolary: If the (left) automaton is epsilon-free, so
59  * is the result
60  *
61  * The alphabets of @c left should meet one of the following requirements:
62  * 1) The output alphabet is the lemma alphabet of the lexicon used
63  * by the language model.
64  * 2) The input alphabet of @c left is the lemma pronunciation alphabet
65  * of the lexicon used by the language model.
66  * In case of ties 1) is chosen.
67  */
