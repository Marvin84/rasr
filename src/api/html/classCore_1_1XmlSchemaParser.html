Detailed Description

Stack based XML parser.

This class should be used for deriving application specific XML parsers. It enhances XmlParser by providing means for reacting according to the current position in the XML element hierarchy.

Design: When writing an event driven parser (e.g. based on the SAX interface, by deriving from XmlParser) you will quickly find yourself introducing state variables and writing complicated callback functions with many case distinction. Following "Design Patterns" (available in library) we apply the State pattern to concentrate the state dependent behaviour and make transitions between states more evident. Participants:

  • Conext: XmlSchemaParser
  • State: XmlElement

Since XML applications are context free languages we have to implement a push-down automaton. For this reason the state of the parser is not defined by a single XmlElement, but by a stack of XmlElements.

Instances of (decendents of) XmlElement can be seen as content models for the various elements of an XML applications. Most predefined content models use method pointers to re-delegate work back to the (applications specific) parser. Note that these callback methods are state dependent; the whole state discrimination and transition business is already take care of.
