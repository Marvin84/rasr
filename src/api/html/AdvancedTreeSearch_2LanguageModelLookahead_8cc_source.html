99 /*
100  * To enhance opportunities for future reuse the look-ahead tree
101  * structure is independent from the HMM state-tree structure.
102  *
103  * Memory consumption of LM look-ahead is determined by two
104  * parameters: The size of the look-ahead tables, and the number of
105  * tables kept im memory.
106  *
107  * The look-ahead table size is determined by the level of detail
108  * reflected in the look-ahead structure. This can be controled the
109  * parameters tree-cutoff and minimum-representation. By default the
110  * look-ahead tables are not pruned, so memory consumption is
111  * (unnecessarily) large.
112  *
113  * The number of tables kept in memory is affected by history-limit,
114  * cache-size-low and cache-size-high.
115  *
116  * By default tables are deleted as soon as they are not immediately
117  * needed (i.e. all relevant trees are pruned). We say, the table
118  * becomes inactive. This saves memory but wastes computation time.
119  * The caching strategy works as follows: As long as there are less
120  * than cache-size-high tables in memory, no table is deleted. Above
121  * that limit the table that has least recently become inactive is
122  * freed whenwever a table is release. Note that cache-size-high is
123  * not a strict upper bound on the number of tables. It will be
124  * exceeded when the search process requires it.
125  *
126  * When a new table is requested, usually one of the inactive tables
127  * is re-used. But when there are less than cache-size-low tables in
128  * memory, a new table is created instead. cache-size-low is thus a
129  * (soft) lower bound on the number of tables in memory. (Of course
130  * there may be less tables, if the search never requires that many
131  * different contexts.)
132  *
133  * Tree cutoff: In the full look-ahead structure non-branching state
134  * sequences are represented by a single node. The MINIMUM depth of
135  * these states is used as pruning criterion: If it is larger than
136  * "network-cutoff", the look-ahead node is merged with its parent.
137  * (This minimum criterion is believed to best reproduce the behavior
138  * of the old standard system. However, other criteria might be
139  * better.)
140  *
141  * TODO:
142  * Optimizations to be considered:
143  * - Different LA table cut-off criteria (may-be)
144  * - Lazy-evaluation of LA tables (may-be; partially done by "lazy lookahead")
145  */

150 // Finds the integer square root of a positive number of any type
151 template<typename type>
152 type isqrt(type remainder) {
153  if (remainder < 0) // if type is unsigned this will be ignored = no runtime
154  return 0; // negative number ERROR
155 
156  type place = (type)1 << (sizeof(type) * 8 - 2); // calculated by precompiler = same runtime as: place = 0x40000000
157  while (place > remainder)
158  place /= 4; // optimized by compiler as place >>= 2
159 
160  type root = 0;
161  while (place) {
162  if (remainder >= root + place) {
163  remainder -= root + place;
164  root += place * 2;
165  }
166  root /= 2;
167  place /= 4;
168  }
169  return root;
170 }
