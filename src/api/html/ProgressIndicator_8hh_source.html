26 /** Provide assurance to impatient users.
27  *
28  * A progress indicators provides a sign of activity during
29  * lengthy operations. The only purpose is to reassure the user
30  * that the machine is still working and will eventually complete.
31  *
32  * This class has been designed for minimal performance penalty.
33  * Unlike simple-minded printf("%d\r", i) approaches, you may
34  * safely use ProgressIndicator::notify() from the inner-most loop
35  * of a highly compute intensive algorithm.
36  *
37  * Usage:
38  *
39  * Usually you will have some way to measure the amount of work
40  * done. E.g. "bytes transferred", "time frames processed", "lines
41  * read". Ideally you know the total number of these "work
42  * units" in advance, or have an upper bound which will be lowered
43  * later on.
44  * -# Create an instance of ProgressIndicator.
45  * -# Call start(). If known, provide the total number units to
46  * process.
47  * -# Call notify() with the current number of units processed
48  * each time you come around. If necessary you can also update the
49  * total. If you have no idea how much work has been done, just
50  * call notify without arguments.
51  * -# When your task is complete, call finish(). This will hide
52  * the progress indicator.
53  *
54  * Limitations:
55  * - Currently there can be at most one progress indicator active
56  * at any time.
57  * - There are no provisions for thread safety.
58  *
59  * TODO:
60  * - Should detect if cout is connected to a TTY, and suppress
61  * messages otherwise.
62  * - see Limitations
63  */

131  /**
132  * Indicate the current state of operation.
133  * You may call notify() very frequently, e.g. from your inner
134  * loop. The run-time penalty of notify() is very small, since
135  * actual updates of the indicator (involving output) are done
136  * only a few times per second.
137  */
138  void notify(u32 done) {
139  if (done > done_)
