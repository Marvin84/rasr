23 /** Packet delay filter.
24  * Inputs: one
25  * Outputs: many, dynamically generated. Port name must be
26  * positive or negative integer corresponding to the desired delay.
27  * Parameters:
28  * -@c max-size is the maximum number of outputs, and
29  * -@c right is the index of the 0 delay from the right side.
30  * Output port 0: no delay, i.e. most recent frame
31  * Output port t: delay of t frames, i.e. each frame is t *
32  * frame-shift older/newer than the current frame on port 0.
33  * Margin cases:
34  * -Parameter @c margin-policy defines how missing frames are generated:
35  * copy: missing frames are generated by copying the closest frame.
36  * mean: missing frames are generated by copying the mean frame.
37  * zero: missing frames are generated by adding 0s.
38  * one : missing frames are generated by adding 1s.
39  * -Parameter @c margin-condition defines when the outputs become valid
40  * not-empty: if at least one of the delays contain a frame. This condition ensures
41  * that each input frame can be retrieved at all delays.
42  * present-not-empty: if the delay=0 contains a frame. This condition ensures that
43  * the number of input frames is equal to the number of output frames.
44  * full: if all the delays contain a frame. This condition ensures that
45  * non of the outputs contains a missing, i.e. a newly generated frame.
46  * Note: Time stamps are not changed.
47  */
