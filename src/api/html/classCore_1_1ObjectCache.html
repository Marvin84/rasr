Detailed Description

template<class List>
class Core::ObjectCache< List >

ObjectCache: implements the caching of larger objects.

Datas are created by the ObjectCache. Datas can be accessed by their keys. Access is devided into readAccess and writeAccess to minimize the number of file operations. If more then maxSize_ object are referenced the last object in the CacheList is written back to its file if necesseary and it is removed from the memory. The last object is given by the cache strategy implemented by the CacheList. The files are stored in one directory. The file names are the keys of the Datas. Datas can be explicitly saved (writeBack) and explicitly removed (erase) from the memory. Store references to Datas with care! References are valid until the next access to the cache.
