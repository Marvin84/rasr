35 /**
36  * @page Configuration
37  *
38  * All resources from
39  * - environment variables
40  * - configuration files
41  * - the command line
42  *
43  * are kept within an @c Configuration object. Each configurable
44  * module (derived form @c Configurable) may ask this class for
45  * resources using a parameter specification string of the form
46  *
47  * <selector1>.<selector2>. ... .<selectorN>.<name>
48  *
49  * A resources consists of a name and an associated value. The name
50  * of the resource has the form
51  *
52  * <selector1>.<selector2>. ... .<selectorN>
53  *
54  * where any of the selectors may by the wildcard "*". The values
55  * are stored as strings. Conversion to appropriate type (flag,
56  * bool, int, float, choice, ...) is done by parameter declaration
57  * classes derive from Parameter.
58  *
59  * @see Core::Configuration
60  * @see Core::Configurable
61  * @see Core::Parameter
62  *
63  * @subsection Resource References
64  * The value of a resource may contain a reference of the form
65  * $(selector). When this resource is looked up (i.e. matched
66  * against a parameter specification), the reference is textually
67  * replaced by its resolved value. The resolved value is determined
68  * by appending the resources selector to the matched parameter
69  * specification and searching for a matching resource. This implies
70  * that resources are @em context @em dependent. If this failed the
71  * matched parameter specification is iteratively truncated until
72  * either a match is found or the resolution fails. Example:
73  * Given the resources
74  * *.foo.abc = cat
75  * *.foo.xyz = dog
76  * foo.*.bar = /tmp/$(foo).txt
77  * then looking up
78  * foo.abc.bar -> /tmp/cat.txt
79  * foo.xyz.bar -> /tmp/dog.txt
80  * @see Core::Configuration::resolve()
81  */
