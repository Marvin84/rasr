Detailed Description

Application class:

The application class provides a suitable wrapper for

  • global initializations
  • configuration
  • C++-like entry and exit points

The default configuration mechanism consists of

  1. preparsing the commandline arguments and look for parameter '–config'
  2. reading the configuration file
  3. parsing the environment variable
  4. final parsing of the commandline arguments

The default name of the configuration file is the application title provided to the constructor concatenated by '.config'. The default name of the environment variable is the capitalized application title.

To build a concrete application, derive from Core::Application and use the macro APPLICATION(ConcreteApplicationClass) to create the class instance and the global entry point. Individual modules have to be initialized in the application class constructor using INIT_MODULE(Module). Initialization is required to register Flow nodes etc.
