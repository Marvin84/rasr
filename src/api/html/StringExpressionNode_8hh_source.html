24 /** String expression resolved by node inputs.
25  *
26  * Usage:
27  * template: 'string1$input(inputPortName1)string2$input(inputPortName2)...'
28  * input ports to read data from will be: inputPortName1, inputPortName2
29  * resulting string value: template expression, but $input(inputPortName1),
30  * $input(inputPortName2), ... replaced by input values
31  *
32  * Example:
33  * ...
34  * <node name="cepstrum" ... warping-function="linear(a=$input(warping-factor))"/>
35  * <link from="..." to="cepstrum:warping-factor"/>
36  * ...
37  *
38  * Value of the expression in warping-function will be resolved for each input at the
39  * port warping-factor.
40  *
41  * Synchronization of ports:
42  * Classes derived from StringExpressionNode will call the update(Timestamp ts) function to
43  * update the string-expression input-ports. Input ports are read if the timestamp 'ts' does
44  * not fall into the intervall [start-time..end-time] of the current parameter value. Thus,
45  * parameters valid for a complete segment should have start-time equal segment start-time and
46  * end-time equals segment end-time. It is of course possible to have parameters valid for
47  * shorter intervals than the complete segment.
48  */
