18 /**
19  * Sprint Unit Testing Framework
20  *
21  * What is a unit test?
22  *
23  * A unit test verifies the functionality of individual units of source code.
24  * A unit is the smallest testable part of an application, in our case a class.
25  * Have a look in the literature for more detailed descriptions of the
26  * principles of unit testing.
27  *
28  *
29  * How to write unit tests?
30  *
31  * - Create a file Test/<Module>_<SourceFile>.cc
32  * e.g. Test/Core_StringUtilities.cc
33  * - Write test cases using TEST() or TEST_F() (see below)
34  * - add the new file to Test/Makefile to the TEST_O target
35  * - run 'make test' in src/Test
36  *
37  * Simple test cases are defined using
38  * TEST(Module, TestSuite, TestCase) {
39  * // test code
40  * }
41  *
42  * More complex tests can be defined using a TestFixture:
43  *
44  * class MyTest : public Test::Fixture {
45  * public:
46  * void setUp() {
47  * // initialize and create test data
48  * // using member variables
49  * }
50  * void tearDown() {
51  * // delete test data
52  * }
53  * protected:
54  * // shared members
55  * }
56  *
57  * TEST_F(Module, MyTest, Foo) {
58  * // test behavior "Foo"
59  * // using the initialized members
60  * }
61  *
62  * TEST_F(Module, MyTest, Bar) {
63  * // another test case using the same data
64  * }
65  *
66  * The individual tests have to use the EXPECT_* macros (see below).
67  * For example
68  * EXPECT_EQ(10, value);
69  * EXPECT_TRUE(doit());
70  *
71  * See Test/Core_StringUtilities.cc for an example.
72  *
73  *
74  * The unit testing framework is based on CppUnit.
75  * See http://cppunit.sourceforge.net/doc/lastest/cppunit_cookbook.html
76  */
