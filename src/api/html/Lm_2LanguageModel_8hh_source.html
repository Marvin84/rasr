203 /**
204  * Abstract language model.
205  *
206  * This class provides the abtract state machine interface used by
207  * all language models in Sprint.
208  *
209  * Design: An abstract state object (of class History) is used to
210  * describe the history (preceeding words). History object can be
211  * used to inquire the probability of the next word (score()),
212  * and they can be extended by a word (extendHistory()). The
213  * (concrete) language model itself specifies what information is
214  * stored in the history descriptor.
215  *
216  * Building your own language model: Mimimally, you have to
217  * implement the three abstract (prue virtual) methods
218  * startHistory(), extendHistory() and score(). Some other
219  * methods only have inefficient default implmentations, you might
220  * have to overload them.
221  *
222  * On sentence boundaries:
223  * In order to fit sentence boundary modelling smoothly into the
224  * n-gram method, one commoly introduces special sentence boundary
225  * tokens. Finite state grammars will normally not do this: Here
226  * the initial state and final scores do the job. For n-gram
227  * models however, there are two common conventions:
228  * A. The nesting-oriented (two symbol) approach uses one token
229  * for the beginning of the sentence and one for the end of the
230  * sentence. Borrowing from XML/SGML syntax these are commonly
231  * denoted as "<s>" and "</s>" respectively. It is silently
232  * assumed that p(<s> | </s>) = 1.
233  * B. The stream-oriented (one symbol) approach uses only a single
234  * sentence boundary symbol. In XML-ish syntax it should be
235  * called "<s/>", but common symbols are "@", "<s>" (which would
236  * be aprropriate in SGML), and "</s>" (which would be wrong in
237  * both XML and SGML).
238  */
