27 /** Filter bank
28  * Constrains and parameters:
29  * -Each filter is assumed to have the same length.
30  * -Filters have a central point and relative to this a left (negative number) and a right border.
31  * Outside of the borders (f < left border and f > right border) filter weights are zero.
32  * Left and right borders are configured by 'normalizedCenterPosition', @see FilterBuilder.
33  * -Filters are placed equidistantly given the spacing.
34  * @param shouldWarpCenterPositions_ controls over which axis the spacing is constant:
35  * -yes : filters are placed equidistantly over the warped axis.
36  * -no : filters are placed equidistantly over the original unwarped axis.
37  * Implemented by simply pre-warping the center positions, since the inverse warping
38  * applied which building the filters will yield equidistant filters over the original unwarped axis.
39  * Note that the filter widths are warped in these cases.
40  * -If spacing is set to zero, it will be set automatically to normalized-center * filter-width,
41  * i.e. the size of the left flank.
42  * -Parts of filters reaching out of the interval [f_min..f_max] are set to zero.
43  * -Positions of first and last filters are controlled by different boundary type classes,
44  * @see FilterBank::Boundary.
45  * Supported types:
46  * -include-boundary
47  * -stretch-to-cover
48  * -emphasize-boundary.
49  */

168  * warping-function: is a string containing a declaration of a function conform to
169  * Math::AnalyticFunctionFactory. Example: nest(linear-2(1.2, 0.875), mel)
170  * This node class is derived from Flow::StringExpressionNode, thus it supports
171  * parameter resolving with incoming values one or more ports. Example:
