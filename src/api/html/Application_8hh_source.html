42 /**
43  * Application class:
44  *
45  * The application class provides a suitable wrapper for
46  *
47  * - global initializations
48  * - configuration
49  * - C++-like entry and exit points
50  *
51  * The default configuration mechanism consists of
52  *
53  * -# preparsing the commandline arguments and look for parameter '--config'
54  * -# reading the configuration file
55  * -# parsing the environment variable
56  * -# final parsing of the commandline arguments
57  *
58  * The default name of the configuration file is the application
59  * title provided to the constructor concatenated by '.config'. The
60  * default name of the environment variable is the capitalized
61  * application title.
62  *
63  * To build a concrete application, derive from Core::Application and
64  * use the macro APPLICATION(ConcreteApplicationClass) to create the
65  * class instance and the global entry point.
66  * Individual modules have to be initialized in the application class
67  * constructor using INIT_MODULE(Module). Initialization is required to
68  * register Flow nodes etc.
69  **/
